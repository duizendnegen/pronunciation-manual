### Q3.2 — How should nginx be configured in Docker Compose?

The `dist/` build output needs to be served. Two approaches:

Option A — inline config via `command:`:
```yaml
services:
  app:
    image: nginx:alpine
    ports: ["3001:80"]
    volumes:
      - ./dist:/usr/share/nginx/html:ro
```

Option B — custom `nginx.conf` file in repo:
```yaml
services:
  app:
    image: nginx:alpine
    ports: ["3001:80"]
    volumes:
      - ./dist:/usr/share/nginx/html:ro
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
```

- [x] Option A — no custom nginx.conf, default nginx static serving is sufficient ← recommended: less files, default nginx handles static HTML fine
- [ ] Option B — custom nginx.conf for explicit control (cache headers, gzip, SPA fallback)
