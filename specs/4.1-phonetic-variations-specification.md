# Epic 4.1 — Many More Phonetic Variations — Specification

## Overview

Expand `frontend/lib/generator.ts` with a much richer set of phonetic substitution rules and creative distortion techniques. Every applicable phonetic rule produces its own variant (instead of stopping at the first match), dramatically increasing pool size and output variety. No new variant styles are introduced — silent-letter / spelling-literal tricks are folded into the existing `phonetic_swap` style as targeted rules. The output contract (3 deduplicated variants, deterministic per input) is unchanged.

## Phonetic rules

### Rule format

Each entry in `PHONETIC_RULES` stays `[RegExp, string, string]` → `[pattern, replacement, hintDescription]`.

All patterns use the `/g` flag (global replace — all occurrences in the word).

### Pool generation change

`makePhoneticSwap` is replaced by `makePhoneticVariants(input: string): Variant[]`.
It iterates every rule; for each rule that matches the lower-cased input it emits one `Variant`. All emitted variants are pushed into the shared pool. The existing fallback (shift first vowel / insert schwa) is kept and emits one variant if **zero** rules matched.

```
for each rule in PHONETIC_RULES:
  if pattern.test(lower):
    emit { text: lower.replace(pattern, replacement), style: "phonetic_swap", hint }
if no rules matched:
  emit vowel-shift / schwa fallback (existing logic)
```

## Creative techniques

The `CREATIVE_TECHNIQUES` array grows from 6 to 15 entries. Each entry is `[CreativeFn, hintString]`.

### Keep existing 6

| Technique | Example on "banana" | Hint |
|-----------|--------------------|-|
| `s + "-ington"` | "banana-ington" | suffix '-ington' added for flair |
| `s + "-y"` | "bananay" | playful '-y' suffix added |
| `s + "-oo"` | "bananaoo" | whimsical '-oo' ending added |
| first vowel doubled | "baanana" | first vowel doubled for emphasis |
| dash inserted at 1/4 position | "b-anana" | stress dash inserted early |
| vowels uppercased | "bAnAnA" | vowels capitalised for comic stress |

> Note: the "vowels capitalised" technique remains despite the decision to drop new capitalisation tricks — it is already in the list and does add mild visual comedy. It is not expanded further.

### New suffix / prefix techniques (5)

| Technique | Example on "banana" | Hint |
|-----------|--------------------|-|
| `"mc" + s` | "mcbanana" | 'mc' prefix for instant silliness |
| `s + "-inski"` | "banana-inski" | suffix '-inski' added |
| `s + "-ola"` | "bananola" | suffix '-ola' added |
| `s + "-inator"` | "banana-inator" | suffix '-inator' added |
| `s + "-eroo"` | "banana-eroo" | suffix '-eroo' added |

### New structural techniques (4)

| Technique | Example on "banana" | Hint | Notes |
|-----------|--------------------|-|-------|
| Reverse the whole word | "ananab" | word reversed | `s.split("").reverse().join("")` |
| Repeat first syllable (first 2 chars + `-` + whole word) | "ba-banana" | first syllable stuttered | Use first 2 characters as proxy for syllable |
| Insert `"-waka-"` at midpoint | "ban-waka-ana" | '-waka-' inserted mid-word | `s.slice(0, mid) + "-waka-" + s.slice(mid)` where `mid = Math.floor(s.length / 2)` |
| Pig Latin | "ananabay" | pig latin | Move leading consonants to end + "ay"; if starts with vowel append "ay" |

#### Pig Latin algorithm

```
if first char is a vowel:
  return s + "ay"
else:
  find index i of first vowel
  if no vowel found: return s + "ay"
  return s.slice(i) + s.slice(0, i) + "ay"
```

## Pool construction & selection

The overall `generateVariants` function logic changes as follows:

```
pool = []

// Phonetic variants: one per matching rule
pool.push(...makePhoneticVariants(input))

// Creative variants: one per technique (single), seeded by hash
for i in 0..CREATIVE_TECHNIQUES.length-1:
  pool.push(makeCreative(input, h + i, false))

// Shuffle pool with seeded Fisher-Yates (existing algorithm, unchanged)

// Pick first 3 with unique text
variants = deduplicate(pool).slice(0, 3)

// Fallback: combined-creative if still < 3
while variants.length < 3:
  variants.push(makeCreative(input, h + 10 + i, true))
```

All rules are equally weighted — no priority tiers. The shuffle ensures rule order does not bias the output.

## Out of scope

- No new `StyleType` values — `spelling_literal` is not added as a distinct style.
- No capitalisation-only tricks (won't affect TTS pronunciation).
- No configurable variant count — always returns 3.
- No LLM integration (deferred to a later epic).
- No changes to UI, audio playback, or E2E tests beyond what naturally passes with richer pool output.
