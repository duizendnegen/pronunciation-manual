# Epic 3.1 ‚Äî Frontend-Only ‚Äî Specification

## Overview

Move the entire application into the browser. The algorithmic variant generator is ported from the Hono backend into the frontend TypeScript bundle. The Hono server, `POST /variants` API route, `Dockerfile`, and all `src/` backend code are deleted. Vite replaces `Bun.build` for bundling and local development. Docker Compose is kept but now runs `nginx:alpine` to serve the pre-built `dist/` directory. The user experience is unchanged: type a word, click "Pronounce", hear variants via TTS.

---

## Tech stack

| Concern | Before | After |
|---------|--------|-------|
| Runtime | Bun + Hono | Browser only |
| Bundler/dev server | `Bun.build` at server startup | Vite |
| Production serving | Bun HTTP server | nginx:alpine via Docker Compose |
| TypeScript types | `bun-types` | `dom` (via tsconfig lib) |
| Backend deps | `hono` | ‚Äî (removed) |

## Changes per file

### `vite.config.ts` (new)

```ts
import { defineConfig } from "vite";

export default defineConfig({
  root: "frontend",
  build: {
    outDir: "../dist",
    emptyOutDir: true,
  },
});
```

### `frontend/index.html`

Change the script tag from the pre-built `/app.js` to the Vite entry point:

```html
<!-- before -->
<script type="module" src="/app.js"></script>

<!-- after -->
<script type="module" src="./app.ts"></script>
```

Vite resolves TypeScript directly in dev mode and bundles it on `vite build`.

### `frontend/lib/types.ts` (moved from `src/types.ts`)

```ts
export type StyleType = "phonetic_swap" | "creative";

export type Variant = {
  text: string;
  style: StyleType;
  hint: string;
};
```

### `frontend/lib/generator.ts` (moved from `src/lib/generator.ts`)

Identical to the current `src/lib/generator.ts`. Update the import at the top:

```ts
// before
import type { Variant } from "../types";

// after
import type { Variant } from "./types";
```

No other changes to generation logic.

### `frontend/app.ts`

Remove all `fetch` / API logic. Import and call `generateVariants` directly. The `VariantsResponse` and `ErrorResponse` interfaces are deleted. The results section is only used for errors, but since generation no longer fails under normal conditions the error path can be removed entirely.

New submit handler:

```ts
import { generateVariants } from "./lib/generator";

// inside DOMContentLoaded:
form.addEventListener("submit", async (e) => {
  e.preventDefault();
  const trimmed = input.value.trim();
  if (!trimmed) return;

  if (playing) cancelSpeech();
  submitBtn.disabled = true;

  const variants = await generateVariants(trimmed);
  currentVariants = variants;
  if (speechSupported && variants.length > 0) {
    startSequence();
  }

  submitBtn.disabled = input.value.trim().length === 0;
});
```

The `results` element and its error rendering can be removed from the HTML and `app.ts` since there is no error surface. The `#results` section and `setError` / `setLoading` helpers are deleted.

### `frontend/index.html` ‚Äî remove results section


### `tsconfig.json`

```json

    "target": "ESNext",

    "moduleResolution": "bundler",

    "skipLibCheck": true

```

`"types": ["bun-types"]` is removed; `"lib": ["ESNext", "DOM"]` added so browser globals (`window`, `SpeechSynthesis`, etc.) are typed correctly.

### `docker-compose.yml` (replaced)

```yaml
services:
  app:
    image: nginx:alpine
    ports:
      - "3001:80"
    volumes:
      - ./dist:/usr/share/nginx/html:ro
```

Run sequence: `bun run build` first, then `docker compose up -d`.

### `e2e/playwright.config.ts`

Update `baseURL`: use `http://localhost:5173` for local dev (Vite default) or keep `http://localhost:3001` for CI (Docker nginx). Recommend an env var:

```ts
baseURL: process.env.BASE_URL ?? 'http://localhost:5173',
```

### `e2e/tests/ui.spec.ts` (rewritten)

The existing tests reference removed elements (`.card`, `.card-text`, `.card-badge`, `.card-hint`, `.card-play-btn`, `#play-all-btn`, `.loading-text`, `POST /variants` route mocks). All of those are gone.

New test coverage:

| Test | What it checks |
|------|---------------|
| Page load ‚Äî heading | `h1` has text "Pronunciation Manual" |
| Page load ‚Äî button disabled | `#submit-btn` is disabled on load |
| Input ‚Äî button enables | typing into `#word-input` enables button |
| Input ‚Äî button disables on clear | clearing input disables button again |
| Submit ‚Äî button re-enables | after clicking "Pronounce", button re-enables once generation resolves |

No route mocking needed (no network calls). No card or audio-button assertions (nothing is rendered to the DOM after submit).

## Out of scope

- Unit tests for `generator.ts` (no test runner beyond Playwright is introduced)
- Deployment (Vercel/Fly.io ‚Äî Epic 5)
- Error handling for generator failures (the algorithmic generator cannot fail on valid string input)
- CORS, API keys, or any server-side concerns (all eliminated)
