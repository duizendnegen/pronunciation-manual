# Retrospective — 20260301-03

**Task:** 3.1 frontend only
**Date:** 2026-03-01
**Team:** solo

---

## Summary

Migrated the entire application from a Bun/Hono backend to a pure frontend bundle served by nginx. The algorithmic generator was moved from `src/lib/generator.ts` to `frontend/lib/generator.ts`, the fetch-based API call in `app.ts` was replaced with a direct `generateVariants()` call, Vite replaced `Bun.build`, and Docker Compose was updated to serve the pre-built `dist/` via `nginx:alpine`. All 5 new E2E tests passed on the first run.

---

## What went well

- Spec was precise about every file change, import path update, and deletion — zero ambiguity.
- The build (`vite build`) succeeded immediately with no config tweaks needed.
- The `app.ts` refactor was clean: removing the fetch/try-catch block and wiring in `generateVariants` directly simplified the submit handler substantially.
- Previous retro lesson about port 3001 orphan containers applied immediately — `--remove-orphans` cleared the stale container before starting nginx.

## What was hard

- `bun` was not on `$PATH` in the tool's shell environment; needed to prefix with `$HOME/.bun/bin/bun`. Harmless workaround, but worth noting.
- The old orphaned docker container (`say-silly-api-1`) blocked port 3001 on the first `docker compose up`. Resolved with `--remove-orphans`.

## What I'd improve

- Add `dist/` to `.gitignore` (it's currently not tracked, but an explicit entry would be cleaner).
- The `e2e/playwright.config.ts` now defaults to `5173` (Vite dev) but CI needs `3001` (nginx). A comment in the config clarifying the two modes would help future readers.
