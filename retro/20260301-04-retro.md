## Summary

Created the GitHub Actions CI/CD workflow (`.github/workflows/deploy.yml`) and the project README. The workflow builds the Vite frontend, runs the 5 Playwright e2e tests against a `vite preview` server, then syncs `dist/` to S3 and invalidates CloudFront on every push to `main`. The session also confirmed the working tree is secret-free. All 5 e2e tests pass against the production build.

## What went well

- The spec was precise enough that implementation was entirely mechanical — two files, no decisions left open.
- Using `vite preview` instead of `bunx serve` kept the workflow self-contained (Vite is already a devDependency) with no extra package download needed in CI.
- Secrets scan was clean; no cleanup required before publishing.
- `dist/` was already in `.gitignore` — the action item from the previous retro was already resolved.

## What was hard

- Port collision during local verification: `vite preview` started on port 5175 (5173/5174 were occupied by a stale dev server), causing tests to hit the wrong server and all 5 to fail. Killing the orphan processes on 5173/5174/5175 and rerunning resolved it immediately.

## What I'd improve

- The workflow could add `--port 5173` to the `wait-on` step comment, or use Playwright's `webServer` config option to make the server lifecycle fully automatic.
- One-time AWS setup steps (A1–A6 in the spec) are still manual. A future epic could add a Terraform module to codify them.
