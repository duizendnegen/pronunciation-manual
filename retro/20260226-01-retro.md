## Per-teammate reflections

### infra

**What went well:** Parallel file creation was efficient — all 7 scaffolding files created in one batch. Dependencies installed successfully on first attempt once bun was available.

**What was hard:** Bun was not installed on the machine, requiring detection and installation mid-task. The lockfile was generated as `bun.lock` (newer bun) rather than `bun.lockb` (older bun), causing a mismatch with the Dockerfile and `.gitignore` that needed fixing later.

**What I'd improve:** Pre-check for required toolchain before starting file creation. Validate that config references (lockfile names, base image versions) match the actual toolchain version in use.

### backend

**What went well:** Spec was detailed with exact types, error messages, and file structure — no ambiguity. Clear file boundaries enabled efficient parallel creation.

**What was hard:** No `tsc --noEmit` available (TypeScript not a project devDependency), so type correctness could only be verified by careful manual review. Received a duplicate task assignment (minor noise).

**What I'd improve:** Add `typescript` as a devDependency so type-checking can run independently of the runtime. Add a local smoke test (`bun run src/index.ts` + `curl /health`) before marking done, if deps are already installed.

### e2e

**What went well:** Spec provided exact test code, making execution fast with no blockers. The infra teammate had already created the `e2e/` directory and `playwright.config.ts`, so only the test file needed writing.

**What was hard:** Nothing significant — the spec was detailed enough to remove all ambiguity.

**What I'd improve:** Validate tests against a running server before marking complete (even just `npx playwright test --list` for syntax). Test specs this detailed could live as a versioned file in the repo rather than only in a task message.

### verifier

**What went well:** Code review was smooth — all implementation files were clean and correct. Docker build succeeded on first attempt after the lockfile fix. 5/6 tests passed immediately. Caught and fixed the Dockerfile `bun.lockb*` vs `bun.lock*` mismatch; added missing `.gitignore` entries for `test-results/` and `.playwright-mcp/`.

**What was hard:** The missing `ANTHROPIC_API_KEY` created ambiguity about whether to block or proceed, requiring a team-lead round-trip. The e2e directory had no `package.json` of its own — Playwright was a root devDependency, requiring the right invocation path to be figured out.

**What I'd improve:** Agree upfront on how to handle secrets for E2E testing (mock the Claude API, use a test-specific key in CI, or split into "no-key" vs "key-required" test suites). Add a `.dockerignore` to exclude `node_modules/`, `e2e/`, `.claude/`, and `specs/` from the Docker build context. The lockfile glob mismatch should be caught at infra setup time, not verification time.
